# engrm-memory

Use Engrm memory deliberately before coding, during coding, and after coding.

What it teaches OpenClaw:

- when to query Engrm before starting work
- how to reuse prior decisions and fixes
- when to use `save_observation`, `create_handoff`, or `capture_openclaw_content` directly instead of waiting for a session digest
- how to make cross-device and multi-agent memory actually useful

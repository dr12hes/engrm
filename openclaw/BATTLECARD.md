# Engrm vs OpenClaw Memory Competitors

## Biggest Threat

The clearest OpenClaw-native competitor right now is Supermemory's OpenClaw plugin.

Why it matters:

- automatic recall before turns
- automatic capture after turns
- positioned as a native OpenClaw memory layer
- lower-friction "it just works" feel than a skill-only approach

Public references:

- https://github.com/supermemoryai/clawdbot-supermemory
- https://docs.openclaw.ai/help/faq
- https://docs.openclaw.ai/automation/hooks

## What OpenClaw Already Gives Users

OpenClaw itself already ships with built-in memory concepts and hookable automation.

That means Engrm is not competing with "no memory at all".
It is competing with:

- OpenClaw's own baseline memory
- automatic plugin competitors like Supermemory

## Where Engrm Wins

Engrm should not try to win on "generic memory".

It wins on:

- shared memory across devices
- shared memory across OpenClaw, Claude Code, and Codex
- startup project briefs
- Delivery Review
- team memory and shared standards later

Core message:

> Engrm does not just remember. It shows whether the work actually matched the brief.

## Current Gap

OpenClaw plugin strength today:

- native connect flow
- startup brief
- session digest capture
- local memory tools
- cloud-backed recall and telemetry

Still improving:

- fuller automatic capture parity with Claude Code
- richer session evidence beyond digest-level summaries
- stronger automatic cross-session reuse without user prompting

## Path To #1

### 1. Own OpenClaw-native shared memory

The free wedge should be:

- OpenClaw-native memory
- cross-device continuity
- cross-agent continuity

### 2. Own accountability

The paid wedge should be:

- Delivery Review
- promised vs delivered
- scope drift
- reopened-after-done sessions

### 3. Be more automatic than the alternatives

Target behavior:

- OpenClaw starts with a useful project brief
- OpenClaw ends with a saved digest
- recall surfaces without prompt babysitting
- the device and agent are visible in Engrm dashboard analytics

### 4. Market the right thing

Lead with:

- "OpenClaw memory across devices"
- "Shared memory across OpenClaw, Claude Code, and Codex"
- "See whether the work actually matched the brief"

Do not lead with:

- "another memory database"
- "just a skill"
- storage quotas

## Immediate Build Priorities

1. OpenClaw cloud sync for captured session digests
2. OpenClaw telemetry so the dashboard shows real OpenClaw usage
3. Cloud-backed recall for startup briefs and search
4. Better automatic capture than digest-only summaries
5. Public site and registry copy that makes OpenClaw first-class

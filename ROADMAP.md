# Roadmap

This roadmap is intentionally short and product-facing. It reflects the highest-value work for making Engrm a trusted shared memory layer for coding agents.

## Near Term

- Publish the repo publicly as `engrm`
- register Engrm in MCP registries and skill directories
- finish branding cleanup from older internal naming
- harden sync visibility, attribution, and chronology behavior

## Agent Support

- maintain deep Claude Code integration
- expand Codex support within the public hook surface where available
- package Engrm as reusable agent skills for Codex, Claude Code marketplaces, and OpenClaw

## Trust Layer

- improve visibility into what Engrm captured and why
- add stronger correction, moderation, and review flows for shared memory
- expand privacy and tenancy safeguards

## Team Product

- review/publish workflows for observations
- dashboards and knowledge-health metrics
- stronger team namespaces and admin controls

## Proof And Moat

- evals showing reduced rediscovery and better session continuity
- richer workpack ecosystem
- stronger feedback loop between memory, retrieval, and prevention
